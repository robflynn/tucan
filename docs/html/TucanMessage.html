Documentation

The TucanMessage interface provides a base message for use on the TUCAN messaging system. By itself it provides absolutely no functionality but instead it must be implemented and extended by a chlid to provide functionality. Ideally most implementation would at least implement the base methods so that those messages could then be passed to other services on the system and actually be handled properly.
TucanRawData getData() raises(TucanException)
The getData() method returns the raw data for the message. What this data is totally depends on the implementation. It could be sound data or text etc... If an error occurs in this method then a TucanException should be raised.

TucanData getDataObject() raises(TucanException)
The getDataObject() method is another way to get access to the data encapsulated in the message. The method will return a data object that can then give the client the message data in various formats like mime, xml, etc... If an error occurs within this method then a TucanException should be raised.
